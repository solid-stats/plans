2. **C1 fires on EVERY cross-module import** (no `index.ts` barrels exist yet): `requests → auth/routes/{authorization,memory,models}`, `ingest → auth/routes/{models,authorization}` + `statistics/{parser-artifact,service/recalculation}`, `public-stats → statistics/{repository/*,parser-artifact,parity-formulas,export/legacy-public-export,bounty/bounty}`, `admin → auth/routes/{models,authorization,memory}`. Caveat: role patterns match by filename suffix (`*routes.ts`, `*service.ts`, `*repository.ts`, `*usecase.ts`, `*controller.ts`); files named by domain (`recalculation.ts`, `commander.ts`, `provenance.ts`) are unclassified and escape C2 — acceptable for the v1 draft, tighten only if the team adopts strict role-file suffixes.
